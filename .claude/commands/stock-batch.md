# stock-batch

**목적:** 여러 종목을 순차적으로 분석하고, 완료 후 비교 요약 생성  
**사용법:** `/stock-batch TICKER1 TICKER2 TICKER3 ...`  
**예시:** `/stock-batch NVDA PLTR TSLA AAPL MSFT 005930`  
**원칙:** 품질 우선 — 순차 실행. 빠른 것보다 정확한 것이 중요하다.

---

## 실행 구조

각 종목을 stock-review 전체 파이프라인으로 하나씩 순서대로 완전히 분석한다.
한 종목이 완전히 끝난 후 다음 종목으로 넘어간다.

```
TICKER1 → [7단계 완료] → TICKER2 → [7단계 완료] → ... → 비교 요약
```

---

## 파이프라인

`$ARGUMENTS`에서 티커 목록을 추출한다 (공백 또는 쉼표 구분).

---

### STEP 1 — 각 종목 순차 분석

`/Users/IAN/.claude/commands/stock-review.md`를 읽는다.

티커 목록의 첫 번째 종목부터 stock-review 파이프라인(STEP 1~7)을 완전히 실행한다.
한 종목이 완료되면 다음 종목으로 넘어간다.

각 종목 시작 시 사용자에게 진행 상황을 보고한다:
```
[2/6] PLTR 분석 시작 (NVDA 완료)
```

종목별 실패 시 오류를 기록하고 다음 종목으로 계속 진행한다. 전체를 멈추지 않는다.

---

### STEP 2 — 비교 요약 생성

모든 종목 분석 완료 후, 각 티커의 `./logs/signals/{TICKER}_{YYYYMMDD}.json`을 읽는다.

`./logs/batch_summary_{YYYYMMDD}.md`를 생성한다:

```markdown
# Batch Analysis Summary — {날짜}
분석 종목: {N}개 완료 / {M}개 실패

## 매매 신호 요약

| 순위 | 티커 | 신호 | 강도 | 신뢰도 | 데이터품질 | GPT총점 | 진입가 범위 | 손절가 |
|-----|------|------|------|--------|-----------|---------|------------|--------|

(신뢰도 높은 순으로 정렬)

## 즉시 진입 가능 종목 (BUY 이상 + 신뢰도 0.6 이상 + GOOD)
[목록 + 진입 조건 + Kelly 비중]

## 관망 종목
[목록 + 관망 이유]

## 제외 종목 (NO_ACTION / 데이터 품질 POOR)
[목록 + 제외 이유]

## 포트폴리오 구성 제안
- 전체 Kelly 비중 합산이 100% 초과 시 비례 축소 계산 포함
- 동일 섹터 집중 시 경고

## 공통 매크로 시그널
[여러 종목 분석에서 반복적으로 나타난 매크로 변수]

## 다음 재분석 일정
[신호 만료일별 정리]
```

---

### STEP 3 — 완료 보고

- 성공/실패 종목 수
- 신호별 분류 결과
- 신뢰도 TOP 3 종목
- 배치 요약 파일: `./logs/batch_summary_{YYYYMMDD}.md`
- 실패 종목 원인
